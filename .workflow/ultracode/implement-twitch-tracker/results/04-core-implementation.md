# Result 04-core-implementation: Core Implementation

## Summary

Implemented the first end-to-end baseline: Drizzle schema and migrations,
config validation, Twitch adapter interfaces, Hono API route groups, modular
worker loops, REST discovery, chat assignment, IRC persistence, Next.js
analytics pages, EventSub subscription reconciliation, OAuth token maintenance,
EventSub event normalization, aggregate rollups, Caddy config, and Docker
Compose.

## Decisions

EventSub webhook receipt lives in API and persists raw events. The worker loop
reconciles desired webhook subscriptions when enabled and normalizes the initial
event set into stream lifecycle state, `channel_events`, and `raids`. REST
discovery has a real fetch adapter and disabled adapter for local
no-credentials mode. IRC uses a project-owned socket adapter so raw lines and
membership commands can be stored directly. Aggregation uses PostgreSQL rollups
into the aggregate tables.

## Risks

The implementation is a verified baseline, not the finished product.
